Chapter 01 · 它是什么

Quire 填补的中间地带

创始人 Playbook 的内容往往有洞见，但排版常在两端摇摆：要么退化成灰蒙蒙的 Word 文档，要么变成过度设计的营销册。 Quire 瞄准中间地带——奶油色画布、单一强调色、衬线层级、有节奏的章节分隔——让 Agent 在没有专职设计师的情况下也能交付可印刷的长文档。

命名由来

Quire（/ˈkwaɪər/）是印刷术语：折好的一叠纸，装订前最小的书帖单元。 这个 Skill 面向 10–80 页、需要「像书一样被阅读」的文档：创始人指南、思想领导力白皮书、内部手册。

不适合的场景

  • 单页简历、落地页 → 用 one-pager 类 Skill
  • 演示型幻灯片 → 用 slides 类 Skill
  • 不足 8 页的短文 → 普通 Markdown 即可
Think 若用户要的是「上台讲」，Quire 不是答案；若用户要的是「坐下来读」，才是。

Quire 与常见做法

同一套内容，不同载体给人的信任感完全不同。下表说明 Quire 在 Agent 工作流中的位置。

Chapter 02 · 怎么用

Agent 的四步工作流

SKILL.md 是入口：触发后按顺序执行，按需加载 references/，不要一次性塞进上下文。

Step 0 · 确认文档类型

长文档（10 页以上）、封面 → 章节分隔 → 正文 → 收尾。四页以内是 brochure，不是 quire。

Step 1 · 选一个 accent

整份文档只用一种强调色，表面占比 ≤ 5%。预设：clay / sage / iris / ink / rust。章节分隔页用浅色 tint，不用满饱和度铺底。

Step 2 · 排页面序列

每 5–7 页正文插入一次视觉重置：章节分隔、拉页引语，或数据锚点页。创始人会扫读；节奏留住他们。

Step 3–4 · 填模板并验收

从 assets/templates/ 复制骨架，替换占位符；跑 scripts/check.py 与 scripts/build.sh 导出 PDF。

Note 交付前核对：偶数页、accent 不过量、每 spread 最多 2 个 callout、字体已嵌入（pdffonts）。
